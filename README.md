pAtChWoRK builds on components developed in the NextProcurement Project (Open Harmonized and Enriched Public Procurement Platform, European Commission). It uses NLP and large language models to extract, enrich, and explore procurement data, supporting administrators, auditors, and bidders to find and analyze information through semantic search, contract classification, and policy insights.

# Deployment Overview

The application is deployed in two independent stages:

- **Backend**: handles data processing, NLP pipelines, and API services.
- **Frontend**: provides the user interface and communicates with the backend.

Both components must be deployed separately. The sections below describe the installation process for each.

# Backend Installation

The **backend** is composed of several services orchestrated via **Docker Compose**:

- **np-solr-api**: REST API for querying and managing procurement data stored in Solr.
- **np-tools**: NLP service that handles enrichment tasks using large language models (requires GPU).
- **solr** + **zookeeper**: Search engine and cluster coordinator for storing and indexing data.
- **solr-config**: Initializes Solr collections and configuration on first run.

---

## 1. Data directory

The backend services expect a shared `data/` directory at the root of the repository. Make sure it exists before starting:

```bash
mkdir -p data
```

---

## 2. Running the backend containers

From the `backend/` directory, start all services with Docker Compose:

```bash
cd backend/
docker compose up -d
```

This will start the following containers:

| Container | Description | Port |
| --- | --- | --- |
| `np-solr-api` | Procurement data API | 9083 |
| `np-tools` | NLP enrichment service | 9084 |
| `np-solr` | Solr search engine | 9085 |
| `np-zoo` | Zookeeper (Solr cluster) | 9086, 9087 |
| `np-solr-config` | Solr schema initializer | 9088 |

---

## 3. Deployment verification

Check that all containers are running:

```bash
docker ps
```

You can also inspect logs for a specific service:

```bash
docker compose logs -f np-tools
```

# Frontend Installation

The **frontend** is an independent project developed in **PHP** using **Bootstrap** for the user interface.
The application is distributed encapsulated within a **Docker container**, allowing the execution environment to be deployed in a simple and reproducible way.

The following steps describe how to install and start the frontend.

## 1. Environment variable configuration

Before starting the container, you need to configure the local paths where the application will store persistent data.

Edit the file:

```
frontend/.env
```

In this file, define the paths to the local directories where the following will be stored:

- **Uploads**: directory where uploaded PDF files will be saved.
- **Data**: directory where the database, extracted text from documents, and other data generated by the application will be stored.

Example configuration:

```env
PROJECT_PATH_UPLOADS=./path/to/your/uploads
PROJECT_PATH_DATA=./path/to/your/data
```

> ⚠️ Make sure the specified directories exist on your machine and have write permissions.


## 2. Frontend dependency installation

The project uses **Node.js** to compile frontend resources (e.g., styles or scripts).

From the corresponding directory, run:

```bash
cd frontend/www/
npm run build
```

This command will install the necessary dependencies and generate the static files required by the application.

## 3. HTTP authentication configuration (Apache)

To protect access to the frontend, you need to create the **Apache** authentication files:

* `.htaccess`
* `.htpasswd`

The `.htpasswd` file will contain the authorized users and passwords to access the application.

To generate passwords, it is recommended to use the official Apache tools, for example:

```bash
htpasswd -c .htpasswd username
```

This command will prompt for the password and generate the corresponding hash.

> You can add additional users by running the same command without the `-c` option.

## 4. Running the Docker container

Once the previous steps are complete, the frontend container can be started using **Docker Compose**.

```bash
cd frontend/
docker compose up -d
```

This command will:

* Build the container image (if necessary)
* Start the service in the background
* Mount the defined volumes to persist data


## 5. Deployment verification

After starting the container, the frontend should be available at the URL configured in `docker-compose.yml` or on the port exposed by the container.

To check that the container is running correctly:

```bash
docker ps
```

You can also check the logs with:

```bash
docker compose logs -f
```
